##  과제 개요

공장 작업자가 **전자저울에서 측정한 무게를 ERP에 손으로 입력**하는 과정을 자동화하는 데스크톱 프로그램을 개발합니다.

저울이 USB-Serial로 보내는 데이터를 PC가 수신 → 무게 숫자만 추출 → 화면 표시 + **클립보드 자동 복사** 하여, 작업자는 ERP 입력란에 무게가 자동출력력 되게 만드는 것이 목표입니다.
(메모장에서도 확인 가능 : Ctrl+V  누르면 저울의 무게가 메모장에 붙여짐)

```
저울 ─(USB-Serial)─> PC → [내 프로그램] → 무게 추출 → 클립보드 → ERP 붙여넣기
```

### 이 과제로 배우는 것
- 시리얼 통신 (pyserial)과 다양한 통신 파라미터(보드레이트/패리티/데이터비트)
- 문자열 파싱 / 정규식으로 무게 데이터 추출
- GUI 개발 (Tkinter)
- 멀티스레드 (수신은 백그라운드, 화면은 멈추지 않게)
- 설정 파일(JSON) 읽기/쓰기, 로깅, 예외 처리
- 하드웨어 없이 테스트하는 방법 (Python 시뮬레이터 / 가상 포트)
- **아두이노로 실물 시리얼 신호 만들기** (하드웨어-소프트웨어 통합 검증)
- 실행 파일(.exe) 배포 (PyInstaller)

##  지원 사양 (요구사항 정의)

아래 제조사 저울을  지원해야 합니다.

 - CAS (CK200SC)

###  아두이노 가상저울 시뮬레이터 — *(하드웨어 실습)
실제 저울 대신 **Arduino UNO**가 진짜 시리얼 신호를 송신하도록 만들어, 내 프로그램이 실물 하드웨어와 동일하게 동작하는지 검증합니다. 

**준비물**
- Arduino UNO (또는 Nano) + USB 케이블
- Arduino IDE

**구현 (`arduino_scale_simulator/` 폴더)**
- [ ] `Serial.begin(9600)` 로 9600bps 송신
- [ ] **저울울 제조사 형식을 출력할 수 있게 (아두이노 코드 개발)
  - CAS:  `ST,GS,   37.39,kg`
> ⚠️ **포트 충돌 주의:** Arduino IDE의 시리얼 모니터가 COM 포트를 잡고 있으면 내 프로그램이 못 엽니다. 업로드 후 시리얼 모니터를 **반드시 닫고** 테스트하세요.

- 현장 배포 PC에  Python이 설치되어 있지 않아도 실행되도록
