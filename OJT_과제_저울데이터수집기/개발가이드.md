# 표준 통합 저울 데이터 수집 프로그램 — 개발 가이드

> OJT 과제 학습 정리 문서
> 목표: 전자저울이 USB로 보내는 무게 데이터를 PC가 받아서 → 숫자만 추출 → 화면 표시 + 클립보드 자동 복사

---

## 전체 그림

```
저울 ─(USB-Serial)─> PC → [내 프로그램] → 무게 추출 → 클립보드 → ERP 붙여넣기
```

핵심 흐름: **"데이터를 어떻게 받느냐(시리얼)" → "어떻게 숫자만 뽑느냐(파싱)" → "어떻게 보여주고 복사하느냐(GUI+클립보드)"**

## 먼저 알아야 할 핵심 용어

### 시리얼 통신 관련

저울은 데이터를 **시리얼 통신(Serial Communication)** 으로 보냅니다. 한 번에 1비트씩 순서대로 전기 신호를 보내는 방식인데, 양쪽(저울과 PC)이 **똑같은 규칙(통신 파라미터)** 으로 맞춰져 있어야 글자가 안 깨집니다.

| 용어 | 뜻 | 이 과제에서 |
|------|-----|------------|
| **Baud rate (보드레이트)** | 초당 신호 전송 속도 | 과제에서 `9600` bps 사용 |
| **Data bits (데이터 비트)** | 글자 하나를 몇 비트로 표현하나 | 보통 8 |
| **Parity (패리티)** | 오류 검출용 추가 비트 | 보통 None |
| **Stop bit (정지 비트)** | 글자 끝을 알리는 비트 | 보통 1 |
| **COM Port** | Windows가 USB 시리얼 장치에 붙여주는 이름 | `COM3` 같은 식. 장치관리자에서 확인 |
| **USB-Serial** | USB를 옛날 시리얼(RS-232)처럼 쓰게 해주는 변환 | 저울 안/케이블에 변환칩이 들어있음 |

> **중요:** 한 COM 포트는 **한 프로그램만** 열 수 있습니다. 아두이노 IDE의 시리얼 모니터가 포트를 잡고 있으면 내 프로그램이 그 포트를 못 엽니다. 테스트할 때 꼭 닫으세요.

### 데이터 형식

CAS 저울(CK200SC)은 이런 한 줄 문자열을 반복해서 보냅니다:

```
ST,GS,   37.39,kg
```

- `ST` = Stable(무게가 안정됨), 흔들리는 중이면 `US`(Unstable)
- `GS` = Gross(총중량), `NT`면 Net(순중량)
- `37.39` = **우리가 뽑아야 할 무게 숫자**
- `kg` = 단위

여기서 `37.39`만 정확히 추출하는 게 **파싱(Parsing)** 단계입니다.

## 개발 순서 (단계별)

추천 전략: **"하드웨어 없이 먼저 다 만들고, 마지막에 진짜 신호로 검증"**

### 0단계 — 환경 준비
- Python 설치 (3.10+ 권장)
- 라이브러리 설치: `pip install pyserial`
- 프로젝트 폴더 구조 잡기

### 1단계 — 시리얼로 데이터 받아보기 (가장 먼저)

`pyserial` 라이브러리로 COM 포트를 열고, 저울이 보내는 줄을 그대로 콘솔에 찍어봅니다.

```python
import serial
ser = serial.Serial('COM3', 9600, timeout=1)
line = ser.readline().decode('ascii').strip()  # "ST,GS,   37.39,kg"
print(line)
```

- **`readline()`** = 줄 단위로 읽기
- **`decode()`** = 바이트(전기신호 원본)를 사람이 읽는 글자로 변환
- "포트 열기 → 한 줄 받기"만 되면 통신의 절반은 끝.

### 2단계 — 시뮬레이터로 테스트 환경 만들기 (하드웨어 없이)

진짜 저울이 없으니 가짜 데이터를 만들어 1단계를 테스트.

- **방법 A — 파이썬 시뮬레이터**: 또 다른 파이썬 스크립트가 `ST,GS,   37.39,kg` 문자열을 계속 시리얼로 내보냄.
- **가상 시리얼 포트(Virtual COM Port)**: PC 안에서 COM3↔COM4를 가상으로 연결(예: `com0com`). 한쪽으로 시뮬레이터가 쓰고, 다른 쪽으로 내 프로그램이 읽음.

> **가상 포트** = 실제 하드웨어 없이 소프트웨어로 만든 가짜 COM 포트 한 쌍.

### 3단계 — 무게 숫자 추출 (파싱 / 정규식)

받은 줄에서 `37.39`만 뽑습니다. **정규식(`re` 모듈)** 사용.

```python
import re
m = re.search(r'(-?\d+\.?\d*)', line)   # 숫자 부분 찾기
weight = m.group(1)  # "37.39"
```

- **정규식** = "이런 패턴의 문자열을 찾아라" 규칙. `\d`=숫자, `\.`=점, `*`=0개 이상.
- 안정(`ST`)일 때만 복사하게 할지 같은 규칙도 여기서 처리.

### 4단계 — GUI 만들기 (Tkinter)

`Tkinter` 는 Python 기본 내장 GUI 라이브러리(추가 설치 X). 무게를 크게 보여주는 창, 포트 선택, 시작/정지 버튼 등.

- **위젯(Widget)** = 버튼/라벨/입력칸 같은 화면 부품
- **이벤트 루프(`mainloop`)** = 창이 계속 떠 있게 유지하는 반복문

### 5단계 — 멀티스레드 (화면이 안 멈추게)

`readline`은 데이터가 올 때까지 **멈춰서 기다립니다(blocking)**. GUI랑 같은 곳에서 돌리면 창이 얼어붙음. 그래서:

- **메인 스레드** = 화면(Tkinter)
- **백그라운드 스레드** = 시리얼 수신 (`threading` 모듈)

용어:
- **스레드(Thread)** = 한 프로그램 안에서 동시에 도는 작업 흐름
- **블로킹(Blocking)** = 작업이 끝날 때까지 다음으로 안 넘어가고 기다리는 것
- 두 스레드가 데이터를 주고받을 땐 **Queue(큐)** 를 쓰면 안전.

### 6단계 — 클립보드 자동 복사 (이 과제의 핵심 기능)

추출한 무게를 클립보드에 넣으면, 작업자가 ERP/메모장에서 `Ctrl+V`로 붙여넣음.

- 방법 1: Tkinter 내장 (`clipboard_clear()` / `clipboard_append()`)
- 방법 2: `pyperclip` 라이브러리 (`pip install pyperclip`, 더 간단)

### 7단계 — 설정 파일 / 로깅 / 예외 처리

- **설정 파일(JSON)**: 포트/보드레이트 등을 코드에 박지 말고 `config.json`에 저장 → 다음 실행 때 읽어옴 (`json` 모듈)
- **로깅(Logging)**: `print` 대신 `logging` 모듈로 동작 기록을 파일에 남김 → 현장 문제 추적용
- **예외 처리(Exception Handling)**: `try/except` 로 "포트 없음", "케이블 빠짐" 상황에서 프로그램이 죽지 않게

### 8단계 — 아두이노로 실물 신호 검증 (하드웨어 실습)

진짜 시리얼 신호로도 잘 되는지 확인. Arduino UNO에 저울 흉내 코드를 올림.

```cpp
void setup() { Serial.begin(9600); }
void loop() {
  Serial.println("ST,GS,   37.39,kg");
  delay(1000);
}
```

- **Arduino IDE** = 아두이노에 코드를 작성/업로드하는 프로그램
- **스케치(Sketch)** = 아두이노 코드
- 업로드 후 **시리얼 모니터 꼭 닫기** (안 그러면 내 프로그램이 포트 못 엶)

### 9단계 — exe로 배포 (PyInstaller)

현장 PC엔 Python이 안 깔려 있어도 돌아가야 하므로 `.exe` 하나로 묶음.

```
pip install pyinstaller
pyinstaller --onefile --windowed main.py
```

- `--onefile` = 파일 하나로
- `--windowed` = 콘솔창 안 뜨게(GUI 앱용)

## 추천 진행 순서 한 줄 요약

**환경준비 → (1)시리얼 수신 → (2)시뮬레이터로 테스트 → (3)파싱 → (4)GUI → (5)멀티스레드 → (6)클립보드 → (7)설정/로그/예외 → (8)아두이노 검증 → (9)exe 배포**

처음엔 (1)+(2)+(3)을 콘솔에서 먼저 완성해서 "데이터 받아서 숫자 뽑기"가 되는 걸 확인하고, 그 다음에 GUI를 입히는 것을 권장. 문제가 생겨도 원인을 좁히기 쉬움.
